# מדריך ל-Antigravity — איך לעבוד על הפרויקט הזה

היי Antigravity. אתה אחד משני סוכנים שעובדים על הפרויקט **Idle Bank Empire** יחד עם הסוכן של Claude.
המסמך הזה מסביר לך בדיוק מה לעשות. קרא אותו במלואו לפני שאתה מתחיל. **פעל בדיוק לפי ההנחיות.**

## מי אתה ומה תפקידך
אתה מבצע **רק** משימות שהמשתמש (דור) תייג עבורך. Claude מבצע את שלו, אתה את שלך — בלי לדרוך אחד לשני.

## איפה המשימות
כל המשימות נמצאות בקובץ אחד: **TASKS.md**, באזור שנקרא "תיבת קלט"
(כל השורות שבין השורה שמתחילה ב-`=====` ובין שורת ה-`=====` הסוגרת).

כל שורה לא-ריקה שם שאינה מתחילה ב-`#` היא משימה. התגית בתחילת השורה קובעת מי מבצע:

| תחילת שורה | מבצע |
|---|---|
| `ag:` או `antigravity:` | **אתה** |
| `claude:` | Claude — אל תיגע |
| ללא תגית | Claude (ברירת מחדל) — אל תיגע |

➜ **קח רק שורות שמתחילות ב-`ag:` או `antigravity:`. את כל השאר אל תיגע, אל תשנה, ואל תמחק — השאר בתיבה.**

## הכללים (חובה — זהים לשני הסוכנים)

1. **קרא קודם את `CLAUDE.md`** — שם כל כללי הקוד והבנייה של הפרויקט. חובה לכבד אותם. בפרט:
   - אל תערוך קבצים מיוצרים: `app.bundle.js`, `style.css`, `www/`. ערוך את קובצי המקור בלבד.
   - כל מחרוזת טקסט למשתמש נמצאת ב-`locales.js` (עברית היא השפה הראשית).
   - ערכי כסף מעוגלים לסנטים.
2. **חוק מכסה 80%** — לפני כל משימה חדשה, בדוק כמה מהמכסה שלך נצרך. אם הגעת ל-~80% — עצור, שמור מה שעשית, והשאר את השאר לפעם הבאה. אם אין מכסה — אל תעשה כלום, נסה בריצה הבאה.
3. **אל תשבור את הפרויקט** — אחרי כל תיקון הרץ:
   ```
   npm run build
   npm test
   ```
   אם משהו נכשל בגלל השינוי שלך — בטל את השינוי של אותה משימה והשאר אותה פתוחה בתיבה.
4. **אל תעשה `git commit` ואל `git push`.** רק השאר את הקבצים ערוכים.
5. **בסיום משימה מוצלחת** — מחק את השורה מתיבת הקלט, והעבר אותה לחלק "בוצע" ב-TASKS.md עם: תאריך, התגית `ag`, וסיכום של שורה אחת.
6. **כשלא בטוח — דלג ותעד.** עדיף לא לתקן מאשר לשבור.

## דיווח — עמוד 2 בקובץ הוורד
הדוח הוא קובץ Word דו-עמודי: **AGENT_REPORT.docx**. עמוד 1 שייך ל-Claude, **עמוד 2 שייך לך**.
אל תערוך את ה-docx ידנית. השתמש בסקריפט המוכן `append_report.py`, והעבר את הארגומנט `ag`:

1. כתוב את סיכום הריצה לקובץ טקסט זמני (למשל `summary.txt`) בפורמט הזה, כי הסימנים צובעים אוטומטית:
   - משימה שבוצעה: שורה שמתחילה ב-`✔ בוצע: <תיאור>`  → תיצבע **ירוק**
   - משימה שנכשלה/דולגה: שורה שמתחילה ב-`✗ נכשל: <סיבה>`  → תיצבע **אדום**
   - שורת תוצאה: `build: עבר/נכשל | טסטים: עברו/נכשלו`
2. הרץ:
   ```
   python3 append_report.py summary.txt ag
   ```
הסקריפט מוסיף רשומה עם תאריך+שעה בראש **עמוד 2**.

**אם קובץ הוורד פתוח ב-Word** והשמירה נכשלת — זה בסדר: הסקריפט שומר את הרשומה אוטומטית לקובץ תור (`AGENT_REPORT.pending.jsonl`) והיא תתווסף לבד בריצה הבאה כשהקובץ יהיה פנוי. שום דבר לא הולך לאיבוד.
