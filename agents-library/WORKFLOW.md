# מפת תזרים — צוות הסוכנים (גרסה 3)

## שרשרת העבודה הסטנדרטית

```
👤 המשתמש — רעיון / פיצ'ר חדש
        ↓
    🟦 רון (PM)
    קורא קוד → בודק סיכונים → כותב תוכנית → מחלק משימות
        ↓
   ┌────────────────────────────────────────┐
   │        שלב תכנון (מקביל)              │
   │  🟨 אורי   🟨 איתי   🟨 גיל   🟩 נועה │
   │  Blueprint  UI/UX   טקסטים  מיגרציה  │
   └────────────────────────────────────────┘
        ↓
    🔵 רננה (Developer)
    כותבת קוד + מיגרציה + Validation בענף feature/
        ↓
    🟪 עמית (Localization) [מקביל לרננה]
    מתרגם Keys ל-4 שפות
        ↓
    🟥 דניאל (QA) ← חובה, לא מדלגים
    state-checker + edge-tester + qa-reporter
        ↓
    🔶 רפאל (Performance) ← רק אם נדרש
    raf-scanner + memory-inspector + o1-verifier
        ↓
    ✅ אישור מפורש מהמשתמש → Merge
```

---

## שער מסירה — כלל הזהב
**כל סוכן חייב לעבור את ה-🔒 שער מסירה שלו לפני שמעביר.**
שאלה לא ברורה = שואל ולא מנחש.

| סוכן | מתי | מיקום בשרשרת |
|------|-----|--------------|
| 🔵 **נועה** (Migration) | כל פיצ'ר עם שדה State חדש | שלב תכנון, לפני רננה |

| 📊 **מאי** (Analytics) | קצב התקדמות / חשד לנשירה | כל שלב — עצמאי |

| 📣 **גל** (Marketing) | תוכן שיווקי | עצמאי לחלוטין |

## קיצורי דרך — מתי מדלגים על סוכנים

| סוג שינוי | סוכנים נדרשים |
|----------|--------------|
| תיקון באג קטן (ללא State) | רון → רננה → דניאל |
| שינוי טקסט בלבד | גיל → עמית → דניאל |
| שינוי UI בלבד | איתי → רננה → דניאל |
| פיצ'ר עם State חדש | + נועה (חובה) |
| פיצ'ר Early Game | + אדם (מומלץ) |
| גרסה גדולה | כל השרשרת + רפאל (חובה) |
