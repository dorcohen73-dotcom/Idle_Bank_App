## מטרה
בצע סקירת קוד מלאה על השינויים בענף הנוכחי. תחפש תחילה באגים לוגיים ולאחר מכן הערות סגנון. לכל ממצא ציין מספר שורה מדויק.

## שלבי הביצוע

### שלב 1 — הכנה
הרץ את הפקודות הבאות כדי להבין מה השתנה:
```
git diff main...HEAD --stat
git diff main...HEAD
```
אם המשתמש ציין קובץ ספציפי — קרא אותו ישירות.

### שלב 2 — חיפוש באגים לוגיים (עדיפות גבוהה)
בדוק בסדר הבא:

**כלכלה ו-game loop:**
- חישובי הכנסה, ריבית, prestige — האם הנוסחאות נכונות?
- האם `economy-manager.js` מתעדכן נכון עם מנהלים חדשים?
- האם `config.js` מכיל את כל הערכים הנדרשים לפיצ'ר החדש?

**מנהלים ומשימות:**
- האם מנהל חדש רשום ב-`config.js` וב-`economy-manager.js` כאחד?
- האם `mission-controller.js` מטפל נכון בתנאי הפעלה/כיבוי?
- האם מנהל יכול להיות פעיל ללא רכישה (`unlocked` לפני `owned`)?

**שמירה וטעינה:**
- האם שדות חדשים ב-`save-manager.js` מקבלים ערך ברירת מחדל בטוח?
- האם גרסת ה-save עלתה אם המבנה השתנה?
- האם `loadGame` מטפל ב-`undefined` עבור השדות החדשים?

**UI:**
- האם אירוע DOM מחובר ומנותק נכון (`addEventListener` / `removeEventListener`)?
- האם `ui-draw.js` מציג נתונים שיכולים להיות `null` ללא בדיקה?
- האם לוקליזציה חסרה — קיים מפתח ב-`locales.js` לכל שפה (he/en/es/ru)?

**anti-cheat:**
- האם ערכים חדשים מגיעים מה-server-state ולא מהלקוח ישירות?

### שלב 3 — הערות סגנון (עדיפות נמוכה)
לאחר שסיימת עם הבאגים, עבור על:
- שמות משתנים לא עקביים עם שאר הקוד (camelCase vs snake_case)
- פונקציות ארוכות מ-40 שורות שניתן לפצל
- קוד כפול שניתן לשתף
- `console.log` שנשאר בטעות בקוד
