# CLAUDE.md — Idle Bank Empire

Vanilla-JS idle/tycoon game shipped as a PWA (web) and an Android app via Capacitor.

## Architecture

- **Core scripts** load as plain `<script defer>` tags in `index.html`, sharing one top-level scope. Order matters: `config.js → locales.js → economy-manager.js → save-manager.js → mission-controller.js → achievement-controller.js → guard-controller.js → customer-flow-controller.js → prestige-controller.js → game.js`.
- **UI** is an ES-module graph (`app.js → ui/draw|events|tabs/*`) bundled into `app.bundle.js` by `tools/bundle.js` (esbuild, IIFE so `file://` works). Never edit `app.bundle.js` by hand.
- **Styles**: `scss/` compiles to `style.css` via `tools/build-css.js`. Never edit `style.css` directly.
- `game.js` — `IdleBankGame` class: state, tick loop `update(dt)`, guard state machine (`idle → moving_to_teller_N → collecting_from_teller_N → moving_to_vault → depositing`), prestige.
- `save-manager.js` — localStorage save (base64 + checksum), offline earnings, server-time anti-cheat.
- `sw.js` — service worker; `CACHE_NAME` hash is generated by `tools/build.js` from source files. **Changes to core scripts don't reach clients until the `?v=` query param in index.html is bumped and/or `npm run build` regenerates the cache name.**
- `www/` is **generated** by root `build.js` (`npm run cap-build`) — never hand-edit it.

## Build & test

- `npm run build` — bundle + css + SW cache-name hash
- `npm run cap-build` — build + regenerate `www/`; `npx cap sync` / `build_aab.ps1` for Android
- `npm test` — vitest unit tests (`tests/`); `npm run test:e2e` — Playwright
- `npm run lint` — eslint

## Conventions

- Money values are rounded to cents: `Math.round((v + Number.EPSILON) * 100) / 100`
- NaN-defensive style: heal corrupt state fields (default to 0/1) instead of trusting them
- All user-facing strings live in `locales.js` (Hebrew is the primary locale)
- `scratch/` is a gitignored junk drawer; `agents-library/` holds AI team-agent definitions
