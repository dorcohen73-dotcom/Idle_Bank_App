# מפרט: מצב ביצועים לטלפונים חלשים — Claude (מוח) → Antigravity (ביצוע)

> רקע: באפליקציית האנדרואיד (WebView) על מכשירים ישנים המשחק מרצד; בדפדפן תקין.
> הגורמים בקוד: ‎~300 שימושי box-shadow ב-scss, ‏15 קבצים עם @keyframes (חלקם מאנימטים
> box-shadow/filter — יקר מאוד ל-GPU חלש), backdrop-filter ב-12+ מקומות, ו-35 נקודות
> spawnFloating שמייצרות אלמנטי DOM מעופפים. לולאת הרינדור: rAF ב-app.js (שורות ‎~299, 380).

## עיקרון-על
שלושה מצבים: `auto` (ברירת מחדל) / `full` / `eco`. המנגנון: מחלקת CSS אחת על `<body>`
(`perf-eco`) שכל ההקלות נגזרות ממנה, + מדידת FPS חד-פעמית בעלייה שמחליטה עבור `auto`.
אין לפצל לוגיקה בין קבצים — קובץ scss אחד חדש מרכז את כל ההקלות.

## שלב A — תשתית + CSS חסכוני (קומיט 1)

### A1. שדה state חדש
- `state.perfMode: 'auto' | 'full' | 'eco'` — ברירת מחדל `'auto'`.
- להוסיף ריפוי ב-`validateAndHealState` (save-manager.js): ערך לא חוקי → `'auto'`.
- **לא לגעת בקבצי הקונטרולרים של Claude** (guard/customer-flow/prestige/shop) — אין להם קשר.

### A2. מודול `performance-manager.js` (סקריפט core חדש, נטען לפני game.js)
API:
- `PerformanceManager.probe()` — מודד FPS: סופר פריימים ב-rAF במשך 1.5 שניות אחרי טעינת
  המשחק (לא בזמן מסך הטעינה — אחרי שה-UI צויר). מחזיר ממוצע.
- `PerformanceManager.apply(mode, fps)` — קובע את המחלקה על body:
  - `full` → אין מחלקה; `eco` → `perf-eco`;
  - `auto` → `perf-eco` אם fps < 45 **או** `navigator.deviceMemory <= 2` **או**
    `navigator.hardwareConcurrency <= 4` (כששניהם קיימים — deviceMemory לא נתמך בכל WebView).
- לחשוף `window.PerformanceManager.isEco()` לשימוש ה-JS (שלב B).
- לזכור את תוצאת ה-probe ב-state (`state.lastMeasuredFps`) לצורכי דיבוג.

### A3. קובץ `scss/_performance.scss` חדש (מיובא אחרון ב-main.scss — חייב לנצח בספציפיות)
תחת `body.perf-eco`:
- `backdrop-filter: none !important` על כל הסלקטורים שמשתמשים בזה (לרכז כאן, לא לפזר).
- ביטול אנימציות שמאנימטות box-shadow/filter — להחליף ב-`animation: none` או בגרסת
  transform/opacity בלבד (למשל פעימת ה"קבל": scale בלבד, בלי הילה מתרחבת).
- צללים: לא לכבות הכל (יהרוס את העיצוב) — לכבות רק צללים מרובי-שכבות ו-glow
  (`box-shadow` עם 3+ שכבות או blur גדול מ-20px). צל בסיסי אחד נשאר.
- `text-shadow` מרובי-שכבות → שכבה אחת.
- כיבוי `filter: drop-shadow/blur` דקורטיביים.

### A4. חיבור בעלייה
ב-app.js, אחרי שהמשחק מוכן: `probe()` → `apply(game.state.perfMode, fps)`.

**קריטריון קבלה לשלב A:** במצב eco (כפוי ידנית דרך console:
`PerformanceManager.apply('eco')`) המשחק נראה סביר — בלי זוהר אבל לא שבור;
`npm run build` ירוק; כל הבדיקות הקיימות ירוקות.

## שלב B — הקלות JS (קומיט 2)

### B1. דילול חלקיקים
בנקודת המימוש של `spawnFloating` (ui): אם `isEco()` — לרנדר רק 1 מכל 3 קריאות
(מונה מודולו, לא random — דטרמיניסטי לבדיקות). קריאות "חשובות" (רווח שחקן מפעולה
ידנית) תמיד מרונדרות — להוסיף פרמטר `important` ולסמן בקריאות הרלוונטיות.

### B2. האטת ציור לא-קריטי
בלולאת ה-rAF ב-app.js: במצב eco, עדכוני DOM טקסטואליים (מונים, סכומים) רק כל
פריים שני (מונה מודולו 2). את תנועת השומר והלקוחות להשאיר בכל פריים (חלק ויזואלית).

**קריטריון קבלה לשלב B:** במצב eco, ספירת אלמנטים מעופפים ב-DOM קטנה פי ‎~3;
אין רגרסיה בבדיקות; המשחק מרגיש חלק.

## שלב C — מתג בהגדרות + מחרוזות (קומיט 3)

### C1. UI
במסך ההגדרות (איפה שנמצא מתג השפה): בורר שלושה מצבים "מצב ביצועים":
אוטומטי / מלא / חסכוני. שינוי → עדכון `state.perfMode` + `apply()` מיידי + save.

### C2. locales.js
מפתחות חדשים בכל השפות (he ראשית, en/es/ru): `perfModeTitle`, `perfModeAuto`,
`perfModeFull`, `perfModeEco`, וטקסט טולטיפ קצר "מומלץ לטלפונים ישנים".

### C3. בדיקות
- יחידה: ריפוי perfMode לא-חוקי; לוגיקת ההחלטה של auto (מוק ל-fps/זיכרון).
- E2E: טעינה עם `?perf=eco` (לתמוך בפרמטר URL לכפיית מצב — שימושי גם לדיבוג),
  ואימות ש-body קיבל את המחלקה.

## כללים (כמו תמיד)
1. סוכן אחד בכל רגע. Claude סיים — המקל אצלך עד שתדווח.
2. קומיט נפרד לכל שלב, עם `npm run build` (כולל verify) ירוק לפני כל קומיט.
3. לא נוגעים ב: guard-controller, customer-flow-controller, prestige-controller,
   shop-controller, ובלוגיקת המשחק ב-game.js. זה פיצ'ר תצוגה/תשתית בלבד.
4. בסיום כל שלב — דיווח קצר לדור; Claude יעשה סבב אימות בסוף (כולל בדיקה חיה בדפדפן).

## מה נשאר בחוץ בכוונה (לא לבצע ביוזמתך)
- מעבר ל-canvas לחלקיקים — שדרוג עתידי, לא עכשיו.
- שינוי עיצוב במצב full — אפס שינוי לשחקנים עם מכשיר חזק.
- וירטואליזציה של רשימות — רק אם המדידות יראו שצריך.
